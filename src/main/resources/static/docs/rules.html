本页为应用规则说明（策略买卖、撮合成本、风控、单股/组合/批量扫描/实盘差异、页面与落库）。 以 application.yml → quant.* 与单股 BackTestEngine 为准；组合/实盘差异见专节。 改配置后需重启，并同步更新本页。

一、统计口径

  • 权益 = 现金 + 持仓市值（成交已含佣金、印花税、滑点与冲击成本）
  • 峰值回撤 = (历史最高权益 − 当前权益) / 历史最高权益
  • 总收益率 = (期末权益 − 初始资金) / 初始资金
  • 胜率 = 盈利的完整开平回合数 / 全部完整开平回合数
  • 综合成本（单票）= Σ(买入成交额 + 买入佣金) / 股数（不含印花税）
  • 成交汇总 tradeStats：买卖次数/股数/手数/成交额、费用合计、总盈亏(=期末−初始)

二、买入何时触发

  • 金叉首开：上一根 MA5≤MA20，本根 MA5>MA20（收盘确认；单股引擎有效回测通常需 K 线约≥65 根）
  • 必过过滤：RSI14 < 60；ATR14 > 0.001； 硬过滤（可关 market-cap-filter-enabled）：模拟市值 < 50 亿 → 直接跳过、不开仓
  • 可选过滤（yml 默认关）： MA60 上行（仅 ma60[i]≥ma60[i-1]，不要求价在 MA60 上方）； 放量 > 20 日均量×1.2； ADX：须 ADX ≥ adx-min(25) 且 ADX ≥ adx-chop-max(20)（低于 chop 视为震荡拒单）
  • 金字塔加仓（默认开）：该票浮盈≥1%（相对综合成本）且 MA5>MA20、总仓≤80%； 不重验 RSI/ATR；仅成交成功后占档（50%→30%→20%）
  • 金字塔切片：相对「目标满仓」50%/30%/20%；若首批不足 1 手但满仓目标≥100 股 → 抬到 100 股
  • 同日约束（按股票）：该票当日已加仓成交 → 不再受理首开；首开成交当日也不会再挂同日加仓
  • 该票已有未成交买单时，不再挂新买单

三、买入数量怎么算

  • 目标满仓股数 = 可用资金 × 单只上限30% × ATR调节 × 账户仓位系数 ÷ 现价，取整到 100 股
  • ATR调节 = baseAtr(0.05) ÷ 当前ATR，再夹到 [0.2, 1.5]
  • 账户仓位系数（峰值回撤）：<15%→1.0；≥15%且<25%→0.5；≥25%→0
  • 总仓：持仓市值 + 拟买入额 ≤ 总权益 × 80%
  • ATR 缺失（单股引擎）：NaN 时回退为 baseAtr(0.05)；组合/实盘见差异节（NaN→0，通常导致不开仓）

四、卖出何时触发

  • 死叉：挂卖 → 下一有效撮合卖出该标的全部持仓（含今仓，不受 T+1 限制）；已挂卖不刷新信号日
  • 同日优先级（回测）：先止损/trail；若当日已止损清仓（stoppedOutToday）→ 忽略同日死叉
  • 止损线 = max(成本−2×ATR, 成本−权益×2%/股数)，只上移
  • T+1（仅止损/trail）：仅 openDate < 今日 的老仓可卖；可部分卖出（只卖可卖手数）；未全清时保留今仓并重算止损线（单股引擎）
  • 止损跳空成交价：触及后基准价 = max(止损价, 当根最低价)，再套卖滑点/冲击——低开时按止损价成交（偏乐观）
  • 移动止盈：盘后 trail = 持仓最高 − 1.5×ATR（只上移）；次日盘中老仓最低价触及判定
  • 跌停：相对昨收；挂卖连续 3 个交易日失败后按跌停价×0.99 强平
  • 回撤熔断：峰值回撤≥25% → 挂清仓且禁新开；本轮粘性（回测结束或进程重启前不自动解除）

五、撮合与成本

  • 日K/周月（默认 next-bar-open-fill=true）：收盘信号 → 下一根开盘价
  • 分钟序列：开仓/死叉在信号日次日且 bar≥09:45 的第一根可用 K
  • next-bar-open-fill=false：买/卖按当根收盘价撮合（绕过次日开盘逻辑）
  • 静默（仅分钟序列挂新开信号时生效；日 K 无静默）：09:30–09:45、14:45–15:00
  • 撮合步内顺序：先处理卖单，再处理买单
  • 例外：老仓止损/trail 按触及价当根卖
  • 买单超过信号日 +5 日历日未成交 → 取消
  • 买单过滤未成交时挂单保留：熔断/静默/开仓过滤等仅记 REJECT，不清空挂单，可跨 bar/跨日重试，直至成交、现金不足取消、止损清仓取消或过期
  • 佣金万三；印花税千一（仅卖）；无最低佣金 5 元规则
  • 分级滑点（近20均量）：≥2000万→0.05%；≥500万→0.2%；否则→0.5%
  • 冲击：min(0.1×本笔量/20日均量, 2%)；买上浮、卖下浮，与滑点叠加
  • quant.slip-point：仅做入参合法性校验（≥0），不参与成交定价；真实滑点以分级滑点为准

六、开仓过滤与账户风控

  • 涨跌停（相对上一交易日收盘；分钟序列不取相邻 bar）： 主板 ±10% / 创科 ±20%； 软阈值：收盘触及昨收×(1±限幅×0.95) 即判； 或 OHLC 均贴齐限价 ±0.01（封板）
  • 停牌量≤0 → 禁止开仓；持仓复牌无「首日必卖」专项规则
  • 低流动性：近20均量 < min-avg-volume20（演示默认 1000）→ 不开
  • 模拟流通市值 = 现价 × 流通股本(亿股)；未配置时启发式股本： 代码 6*→120、3*→80、688*→40、其它→100； 可用 quant.float-shares-yi.<code> 覆盖；阈值默认 50 亿
  • 单日亏≥昨收权益的 3% → 当日禁新开
  • 连亏 5 笔完整开平 → 当日禁开、次日自动恢复
  • 回撤≥15% 仓位×0.5；≥25% 禁开+清仓挂单

七、买单/卖单未能成交的常见原因

  • 账户熔断禁开 / 单日亏损禁开 / 连亏禁开
  • 开盘静默 09:30–09:45 禁止新开成交（分钟路径）
  • 开仓过滤未过：涨跌停（含 0.95 软阈值）、停牌、流动性、市值、收盘静默等——此时挂单仍在，可稍后重试
  • 加仓时涨停或停牌（单股）；组合/实盘加仓另见差异节
  • 现金不足，或买入后突破总仓 80% → 取消挂单
  • 仓位系数缩放后不足 1 手（100股）→ 取消挂单（单股）
  • 卖出遇跌停：累计失败，满 3 个交易日后强平
  • 买单超过信号日+5 日历日 → 过期取消

八、单只回测（页面「个股回测」）

  • 默认周期 DAY；起止时间留空 = 该股全量可用 K 线
  • 初始资金默认 100000（可改）
  • K 线 < 65 根 → 返回空结果
  • 决策顺序：①撮合挂单（先卖后买）→ ②老仓止损/trail → ③账户风控 → ④收盘挂金叉/金字塔/死叉 → ⑤盘后更新 trail
  • 首开/金叉买单成交：走完整 canExecuteOpenFill；加仓成交：仅拦涨停/停牌
  • 成交股数 = 挂单股 × 当前仓位系数 后再取整手
  • 结果含：权益曲线、买卖点、成交明细、tradeStats、决策分析事件
  • 历史与分析一一对应，写入 MySQL：bt_backtest_record / bt_backtest_analysis

九、组合回测（多股共享资金池）

  • 勾选多只成分股；共用一份现金与同一套账户风控（熔断/连亏/单日亏/仓位系数）
  • 强制用 DAY 行情；按各股交易日并集时间轴推进；某股当日无 K 则跳过该股
  • 单股 K 线不足约 65 根则不进入组合
  • 每只股票独立：持仓 lot、金字塔档位、挂单、止损/trail、stoppedOutToday
  • 买入手数仍受单只30%与总仓80%约束（相对组合总权益；挂单时已用仓位系数算目标量）
  • 与单股差异：
    • 加仓成交也走完整 canExecuteOpenFill（流动性/市值/静默等），不只拦涨停/停牌
    • 成交时不再按仓位系数二次缩量（与单股不同）
    • ATR NaN → 0（通常无法开仓）
    • 部分止损后不重算止损线（单股会 raiseStopByCost）
  • 熔断时：对所有仍有持仓的成分股挂清仓
  • 分股表现：按该股已实现盈亏、买卖统计、该股维度回撤等汇总；贡献率≈该股已实现盈亏/初始资金
  • 历史/分析同样落库（kind=PORTFOLIO）

十、批量扫描（个股回测页「扫描」）

  • 对配置标的列表（默认 600036/000001/300059）线程池并发（池大小 batch-pool-size，默认 10）
  • 每只：全量 DAY K 线 + 初始资金 100000 跑一遍个股回测引擎
  • K 线 < 20 根 → 跳过该股；20–64 根会进列表，但引擎仍要求约≥65 根才有有效回测结果（中间区间多为空结果）
  • 额外计算最新指标与「当前是否金叉可买」信号描述，按总收益率降序展示
  • 扫描结果为即时列表，不单独写入回测历史表（除非你再点个股回测保存）

十一、实盘分钟扫描（定时任务，模拟现金）

  • 默认各任务关闭：配置在 MySQL sys_schedule_job（种子 enabled=0）；页面「定时任务」可启停/改 cron，立即重载
  • 总闸 quant.schedule.enabled（默认 true）：为 false 时不注册任何触发器；目录见 GET /api/schedule
  • 唯一目标池：全市场经 pool-rebuild 或 after-market-batch-scan 盘后扫描自动覆盖 trade_pool；无需人工确认；扫描无入选时清空池
  • pool-rebuild 与 after-market-batch-scan 启用其一即可
  • 页面「交易目标池」可手动移出标的（移出≠卖出，不停仓）
  • 交易时段约每分钟扫描目标池内活跃标的（工作日 9–11、13–15 点）；模拟初始现金默认 100000
  • 规则意图对齐回测：金叉/金字塔/死叉、止损/trail、账户风控、09:45 撮合、跌停强平等
  • 已知差异：
    • stoppedOutToday 未像回测那样在止损后置位，极端情况下同日仍可能再挂死叉
    • 加仓成交走完整开仓过滤（同组合）
    • ATR NaN → 0
    • RiskControlService#checkBuy 另用墙钟交易时段（含静默），并校验持仓后单票市值 ≤ 权益×30%×仓位系数
    • 网关持仓与内存账本可能双路径；sdk 模式为桩实现
  • trade-mode=sim：下单即时成交；sdk：SUBMITTED 后由同步推进（桩）
  • 实盘持仓/现金默认在进程内存，不落 trade_positions（表已建，供后续扩展）
  • 收盘后任务：约 15:30 相关清算；16:00 可触发批量扫描日志等

十二、行情与页面产品规则

  • 行情真相源：MySQL market_daily（日线）+ market_minute（5分钟）； 15/30/60 分由分钟聚合；周/月由日线再聚合；MIN_1 请求降级为 5 分钟序列
  • 聚合交易分钟：约 09:30–11:29、13:00–14:59；周线键为周一所在周
  • 空库启动时从 classpath:data/kline 的 DAY/MIN_5 JSON 导入一次，运行时不再读 JSON 做展示
  • 行情页展示 / K 线图表：走统一 MarketDataService#getKline
  • 回测时间留空 = 全量；填写则截取区间；初始资金默认 100000
  • 侧栏：一级菜单互斥；回测成功前可隐藏权益/成交等结果面板
  • 可选：API Key（QUANT_API_KEY）、回测类接口限流（默认 30 次/分钟/IP）

十三、单只回测 bar 内决策顺序（摘要）

  1. 撮合已挂单（先卖后买；日K开盘 / 分钟次日≥09:45；或 next-bar-open-fill=false 时当根收盘）
  2. 仅老仓止损 / 移动止盈（T+1 分档；跳空价 max(止损, low)）
  3. 账户风控快照
  4. 收盘挂金叉 / 金字塔 / 死叉
  5. 盘后更新最高价与 trail
