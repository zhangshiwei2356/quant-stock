在页面上启停预置任务、修改 cron / 固定间隔；配置写入 MySQL，改完立即重载，无需改 yml 重启。

说明怎么管

  • 表 sys_schedule_job：种子任务默认全关
  • 已实现：盘后入池扫描（覆盖唯一目标池）、分钟扫描（只打目标池）
  • 未实现（待外部 API）：收盘清算、订单同步、行情采集、持仓盈亏同步、数据校验
  • 链路：全市场扫描 → 自动覆盖唯一目标池 → 分钟扫描；pool-rebuild 与 after-market-batch-scan 启用其一即可
  • 总闸 quant.schedule.enabled：false 时不注册触发器，库表仍可编辑

步骤建议流程

  1. 确认 MySQL 已建库且 quant.db-enabled=true
  2. 进入管理页，只打开需要的任务并核对 cron
  3. 多实例部署时依赖 Redis/本地锁，避免重复执行

再点一次「定时任务」可收起并回初始化页。
