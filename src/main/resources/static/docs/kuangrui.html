二、MDS 行情系统能力

入口类：MdsClient（TCP）/ MdsUdpClient（组播）；回调 MdsCallBack / MdsUdpCallBack。

能力典型接口 / 回调对本应用价值
行情订阅推送subscribeMarketData → L1 股票/指数/期权快照实盘扫描、盘中价
Level2onMktL2StockSnapshot*、onMktL2Trade、onMktL2Order、最优档等深度/逐笔；资金流可自算近似
快照查询qryMktDataSnapshot / qrySnapshotList补洞、启动拉全量
证券实时状态qrySecurityStatus / onSecurityStatus停复牌、状态过滤
交易时段状态qryTrdSessionStatus / onTrdSessionStatus开市/休市/集合竞价
证券静态信息qryStockStaticInfo / List（含涨跌停价、停牌标识等）替换启发式市值/涨跌停
期权静态qryOptionStaticInfo*本应用 A 股现货策略暂不需要
UDP 组播MdsUdpClient低延迟；部署与权限要求更高

MDS 通常不直接提供：历史日线/分钟 K 线落库、复权因子、PE/ROE 财报、新闻舆情。回测用历史 K 线仍需另源或本地库。

三、OES 交易系统能力

入口类：OesClient；通道分工：委托 OesOrdChannel、回报 OesRptChannel、查询 OesQueryChannel；回调 OesCallBack.onOrd / onRpt。

能力典型接口对本应用价值
下单 / 批量下单sendOrdReq / sendBatchOrdsReq替换 trade-mode=sdk 桩
撤单sendOrdCancelReq本应用尚无真实撤单链路
回报同步initRptSync / sendRptSync + 委托/成交确认真实成交推进订单状态
查委托 / 成交queryOrder / queryTrade对接 sync-orders
查资金 / 持仓queryCashAsset / queryStkHolding对接 position-pnl-sync
客户总览 / 柜台资金queryClientOverview / queryCounterCash账户与柜台对齐
佣金费率queryCommissionRate替换本地固定费率假设
现货产品信息queryStock（含停牌、风险等级等）硬过滤补充
交易日 / 市场状态queryTradingDay / queryMarketState替代静态节假日表一部分
银证转账sendCashTrsfReq + 流水查询资金出入金（真仓运维）
两融 / 期权 / ETF / 新股中签大量 Crd* / Opt* / Etf* / LotWinning超出当前现货模拟范围，可不接

四、与本应用对照（缺什么）

本应用模块现状宽睿可补仍缺（非宽睿或未覆盖）
market-collect / 行情db/mock 骨架MDS 订阅+快照→落库分钟/日线（需自聚合成 K）历史全市场回测 K、复权
TradeGatewayServicesim 即时成交；sdk 状态桩OES 下单/撤单/回报本地风控与柜台拒单映射
sync-orders未实现queryOrder/queryTrade+回报—
position-pnl-sync未实现queryCashAsset/queryStkHolding—
停复牌 / ST名称粗滤、量≤0MDS 证券状态 + 静态信息；OES queryStock权威 ST/退市名单（可部分覆盖）
交易日历静态节假日queryTradingDay / 时段状态多年历史调休补班表
涨跌停价规则估算MDS 静态上下限价—
佣金成本配置近似queryCommissionRate—
复权 / 资金流 / 财报 / 舆情未落地L2 可近似资金流；其余基本不覆盖需其他数据源

建议接入顺序：① MDS 快照/订阅打通 market-collect → ② OES 下单+回报替换 sdk 桩 → ③ 委托/持仓对账任务 → ④ 静态信息与交易日增强过滤。两融/期权/ETF 暂缓。

五、对接前置条件（非代码能补）

  1. 券商/宽睿侧开通 OES、MDS 账号与通道地址（配置见 *_api_config.json）
  2. 将 quant360-*-api-*.jar 及依赖纳入本工程（或私服）；注意与 Spring Boot / Netty 版本兼容
  3. 实现长连接生命周期：登录、心跳、断线重连、回报续订（demo 可参考 OesExample / MdsExample）
  4. 明确环境：仿真 vs 生产；clEnvId、密码加密方式与柜台一致
